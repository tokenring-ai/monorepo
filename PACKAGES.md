# Token Ring Package Index

## Overview

The Token Ring monorepo hosts the development of the Token Ring Coder, Writer, and WebTerminal Applications, an interactive AI-powered developer assistant. This framework enables conversational interaction with codebases, supporting tasks like code editing, refactoring, testing, git operations, and integrations with external services (e.g., AWS, Docker, web search). Built as a TypeScript monorepo using Bun, it provides pluggable packages under the `@tokenring-ai/*` scope for modular AI agent functionality. Key features include persistent chat sessions in SQLite, command-based workflows, plugin extensibility, and sandboxed execution. The ecosystem targets developers seeking AI-assisted coding in a secure, local environment (version 0.2.0, actively developed).

## Categories

### Core Foundation

- **@tokenring-ai/agent**: Agent orchestration and state management.
- **@tokenring-ai/ai-client**: AI model registry and chat/embeddings.
- **@tokenring-ai/app**: Base application class with service registry.
- **@tokenring-ai/utility**: Shared utilities (cache, logging, shell escape).
- **@tokenring-ai/filesystem**: Abstract filesystem operations.
- **@tokenring-ai/memory**: Short-term memory and attention storage.
- **@tokenring-ai/queue**: Task queuing with checkpoints.
- **@tokenring-ai/checkpoint**: Checkpoint service abstraction.
- **@tokenring-ai/rpc**: RPC Abstraction layer for connecting various RPC interfaces to internal functions.
- **@tokenring-ai/lifecycle**: Lifecycle management for agents and services.

### Storage & Database

- **@tokenring-ai/database**: Abstract DB layer.
- **@tokenring-ai/mysql**: MySQL integration.
- **@tokenring-ai/s3**: S3 filesystem/CDN.
- **@tokenring-ai/drizzle-storage**: Drizzle ORM-based storage.
- **@tokenring-ai/cdn**: Abstract CDN service.
- **@tokenring-ai/browser-storage**: Browser-based state storage.

## Cross-Package Interactions

The ecosystem is agent-centric: `@tokenring-ai/agent` is the hub, registering tools/commands/services from other packages via registries. For example:

- **Agent → AI Client**: All agents use `@tokenring-ai/ai-client` for LLM calls
- **Agent → Filesystem**: Code operations use `@tokenring-ai/filesystem` with provider implementations
- **Agent → Memory**: Context management via `@tokenring-ai/memory`
- **Agent → Queue**: Deferred execution via `@tokenring-ai/queue`
- **Agent → Testing**: Validation via `@tokenring-ai/testing` with auto-repair
- **Agent → Git**: Version control via `@tokenring-ai/git`

### Publishing Workflow

```
TokenRing Writer (app)
  ├─ @tokenring-ai/template (content scaffolding)
  ├─ @tokenring-ai/ai-client (content generation)
  ├─ @tokenring-ai/blog (content abstraction)
  ├─ @tokenring-ai/ghost-io or @tokenring-ai/wordpress (publishing)
  └─ @tokenring-ai/web-host or @tokenring-ai/cdn (delivery)
```

### Financial Data Workflow

```
Agent
  ├─ @tokenring-ai/cloudquote or @tokenring-ai/kalshi or @tokenring-ai/polymarket (market data)
  ├─ @tokenring-ai/research (analysis)
  └─ @tokenring-ai/scripting (trading logic)
```

### Dev Pipeline

- `@tokenring-ai/code-watch` detects changes
- Triggers agent with `@tokenring-ai/javascript` (ESLint, package management)
- Runs `@tokenring-ai/testing` (tests)
- Auto-commits via `@tokenring-ai/git` if passing

### Communication Channels

- `@tokenring-ai/cli`: Terminal UI
- `@tokenring-ai/slack` / `@tokenring-ai/telegram` / `@tokenring-ai/discord`: Messaging platforms
- `@tokenring-ai/web-host`: Web API
- `@tokenring-ai/email` / `@tokenring-ai/calendar`: Communication and scheduling

### Scheduling

- `@tokenring-ai/scheduler` runs agents at intervals
- `@tokenring-ai/queue` processes queued tasks
- `@tokenring-ai/checkpoint` persists state across runs
